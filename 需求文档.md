# 函数变换解码器 - 需求文档

> 一个面向高中数学的交互式教学网页：从基本函数出发，自动拆解学生输入的目标函数的每一步变换，并用丝滑的动画逐步演示变换过程。

---

## 1. 项目概述

### 1.1 项目名称
**函数变换解码器（Function Morph Decoder）**

### 1.2 项目目标
帮助高中生直观理解函数图像变换（平移、伸缩、翻转）的本质。学生输入一个目标函数后，网页自动从基本函数（`y=x`、`y=x²`、`y=x³`）出发，逐步"解码"出每一步变换，并以电影般的动画演示从基本函数演变为目标函数的全过程。

### 1.3 核心卖点
- **自动拆解**：解析任意目标函数，反推出变换链条（顺序正确）。
- **丝滑动画**：图像不"闪现"，而是分步、平滑地平移/伸缩/翻转，配合缓动函数。
- **科技美学**：深色模式 + 霓虹荧光曲线 + 磨砂玻璃面板，营造 "Math + Tech" 氛围。
- **教学提示**：每一步配合中文口诀提示（如"左加右减""上加下减"），强化记忆。

## 2. 技术栈

| 类别 | 选型 | 引入方式 |
| --- | --- | --- |
| 结构 | 原生 HTML5 | 单文件 |
| 样式 | Tailwind CSS v3 | CDN |
| 绘图 | 原生 Canvas 2D API | 浏览器内置 |
| 表达式解析 | math.js | CDN |
| 公式渲染 | KaTeX | CDN |
| 动画 | `requestAnimationFrame` | 浏览器内置 |

### 2.1 关键技术约束
- 全部代码输出在**单个 HTML 文件**中。
- 所有外部依赖通过 **CDN** 引入（Tailwind、math.js、KaTeX）。
- 所有动画使用 `requestAnimationFrame`，目标 **60fps**。
- 缓动函数统一使用 **easeInOutQuad**。
- 兼容 **Chrome、Firefox、Edge** 最新版。
- **响应式设计**，适配桌面与平板/手机。
- 代码结构清晰、注释完善（注释使用中文），便于后续扩展。

## 4. 功能需求

### 4.1 基本函数选择
- 三个单选按钮：`y = x`、`y = x²`、`y = x³`。
- **默认选中 `y = x²`**。
- 切换基本函数时，重新基于该基函数解析目标函数。

### 4.2 函数输入
- 文本输入框，支持输入任意函数表达式（如 `-2(x-1)^2+3`）。
- **实时数学公式预览**（KaTeX 渲染输入内容）。
- 输入非法时给出友好错误提示，不崩溃。

### 4.3 变换步骤分解
- 自动识别从基本函数到目标函数的所有变换步骤。
- 以列表形式展示每一步（含中文口诀提示文字）。
- 支持的变换类型：
  - **上下平移**：`y = f(x) + k`
  - **左右平移**：`y = f(x - h)`
  - **上下翻转**：`y = -f(x)`
  - **左右翻转**：`y = f(-x)`
  - **纵向伸缩**：`y = a·f(x)`
  - **横向伸缩**：`y = f(b·x)`
  - **复合变换**：自动按正确顺序拆解。

### 4.4 动画控制
- 按钮：**上一步**、**下一步**、**播放 / 暂停**、**重播**。
- **速度滑块**：0.5x ~ 2.0x。
- 每步变换默认持续 **1.2 秒**（受速度滑块调节）。
- 当前步骤完成后**高亮**对应步骤项。

### 4.5 Canvas 画布
- 绘制笛卡尔坐标系 + 网格线（带刻度）。
- 原函数：半透明灰色虚线显示（参照）。
- 变换过程中：显示原图像残影。
- 当前变换曲线：主色调霓虹色 + 外发光。
- **鼠标悬停**：显示当前坐标与函数值（十字准星 + 数值气泡）。
- 支持鼠标滚轮**缩放**、拖拽**平移**视图（可选增强）。

### 4.6 数据面板
实时显示当前函数的数学特征：
- 当前解析式（KaTeX 渲染）。
- 顶点坐标（二次函数）。
- 对称轴。
- （可按基函数类型显示对应特征，如三次函数的对称中心等。）

## 5. 动画演变机制（核心）

不允许图像直接"闪现"到新位置，而是**分步解构解析式**，像放电影一样逐帧演示。

### 5.1 示例：`y = -2(x-1)² + 3`
拆解为以下动画步骤：

| 步骤 | 名称 | 提示文字 | 动画效果 |
| --- | --- | --- | --- |
| 第一步 | 本体现身 | 绘制标准二次函数 `y = x²` | 荧光线从原点出发平滑绘制出曲线 |
| 第二步 | 左右平移 | "左加右减：x → (x−1)，向右平移 1 个单位" | 曲线沿 x 轴平滑滑行，网格留下淡入淡出移动轨迹 |
| 第三步 | 伸缩与翻转 | "系数为负：开口向下；系数为 2：纵坐标拉伸为原来的 2 倍" | 曲线以顶点为中心，像橡皮筋般丝滑向下翻转并变窄 |
| 第四步 | 上下平移 | "上加下减：+3，向上平移 3 个单位" | 曲线沿 y 轴垂直升起，定格后顶点闪烁微光，标出 (1, 3) |

### 5.2 变换顺序规则
对于形如 `y = a·f(b(x − h)) + k` 的目标函数，按以下顺序演示（与课堂教学一致）：
1. 绘制基函数 `f(x)`。
2. 横向平移（`x → x − h`）。
3. 横向伸缩（`b`）。
4. 纵向伸缩 + 上下翻转（`a`，含正负号）。
5. 纵向平移（`+ k`）。

> 注：第 2/3 步与第 4 步的先后顺序需保证"先内层（x 相关）后外层（y 相关）"的数学正确性。具体拆解策略在开发阶段以二次函数为主路径，其余基函数尽量复用。

### 5.3 缓动
- 所有过渡使用 `easeInOutQuad`：`t<0.5 ? 2t² : 1−(−2t+2)²/2`。
- 每帧通过 `requestAnimationFrame` 推进，按经过时间（`deltaTime`）计算进度，保证不同帧率下时长一致。

## 6. 代码与交付要求

1. 输出**一个完整的 HTML 文件**，所有 HTML / CSS / JS 内联其中。
2. 通过 CDN 引入 Tailwind、math.js、KaTeX。
3. 良好的缩进与**中文注释**（每行关键代码均有中文注释）。
4. 在 Chrome、Firefox、Edge 中均能正常运行。
5. 响应式设计，适配不同屏幕尺寸。
6. 代码模块化组织，便于扩展（如新增基函数、新增变换类型）。

### 6.1 建议的代码模块划分（单文件内逻辑分区）
- **配置区**：颜色、网格、动画时长等常量。
- **数学解析模块**：用 math.js 解析表达式，反推变换参数 `(a, b, h, k)` 及符号。
- **变换链构建模块**：根据参数生成有序的变换步骤数组。
- **Canvas 渲染模块**：坐标系、网格、曲线、残影、发光、悬停准星。
- **动画引擎模块**：`requestAnimationFrame` 主循环 + easeInOutQuad + 播放控制。
- **UI 控制模块**：单选按钮、输入框、按钮、滑块、步骤列表、数据面板事件绑定。
- **KaTeX 渲染模块**：公式预览与数据面板公式。
